## ADDED Requirements

### Requirement: 事件提取结果写入边界
系统 SHALL 允许 AI 事件提取流水线将通过校验的事件、证据、标签和实体关联写入事件知识 schema，并保留提取状态与审计信息。

#### Scenario: 写入事件事实
- **WHEN** 事件候选通过结构化校验、证据校验和安全边界校验
- **THEN** 系统必须写入或更新事件事实，并保存事件类型、标题、摘要、发生时间、地域、状态、提取来源和处理状态

#### Scenario: 写入事件来源证据
- **WHEN** 事件事实来自一个或多个 raw document
- **THEN** 系统必须通过事件来源证据表关联事件和 raw document，并保存证据摘录、证据哈希、来源等级和内容来源类型

#### Scenario: 写入事件标签和实体关联
- **WHEN** 事件候选包含通过校验的标签和实体关联
- **THEN** 系统必须保存事件标签关联和事件实体关联，并记录标签来源、实体关联类型、证据摘录、置信度和匹配方式

### Requirement: 原始文档不直连实体
系统 SHALL 保持 raw document 与实体事实的间接关系，实体关联必须通过事件提取、校验或审核流程生成。

#### Scenario: 从原文提到实体
- **WHEN** raw document 正文中出现公司、板块、政策机构、经济体、联盟组织或其他实体名称
- **THEN** 系统不得让 raw document 直接写入实体关联表，必须由事件提取结果先形成事件事实，再通过事件实体关联表达
